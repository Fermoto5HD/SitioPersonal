APIs

En esta sección te explico cómo usar las APIs que permiten mostrar la información en forma dinámica en este sitio.


Podés utilizarla si querés. No hay problema.

Comenzamos

Para armar una API se necesita tener una buena base de PHP y Codeigniter.

Las APIs permiten realizar consultas desde un sitio para obtener datos y utilizarlos para cierto fin, sea una app alterna, como para el desarrollo de funciones para un proyecto en específico.

Un buen ejemplo de API armado con Codeigniter es "Trenes en Vivo" desarrollado por la SOFSE (Operadora Ferroviaria Sociedad del Estado / Trenes Argentinos). Y un ejemplo de buena implementación de API está en un proyecto colectivo llamado "Lantalks" en el cual sus datos están disponibles en lantalks.com/APIs

En este caso, decidí liberar una API que permite obtener datos del blog y del portfolio.

El uso es totalmente libre, por lo cual agregué el header para el Cross Origin para que su uso sea ilimitado.

En esta primera versión es muy fácil usarlas. Hay 4 métodos, dos para el blog y dos para el portfolio.

  • Con el método "blog" obtenés el listado de los posts publicados.
  • Con el método "blog_post/[id] obtenés un post específico. En caso de no encontrarlo, arrojará un array vacío.
  • Con el método "portfolio" obtenés el listado de los trabajos realizados.
  • Con el método "portfolio_item/[id] obtenés un trabajo específico. En caso de no encontrarlo, arrojará un array vacío.
